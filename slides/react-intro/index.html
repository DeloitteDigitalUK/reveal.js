Break Pages into Components

Each component is responsible for one thing. A component can have child components which each do a specific task.

We'll make the video and play button.

How to make a component?

Every component has a render method, which describes how it looks.

JSX makes it easy to write render methods because it feels like writing HTML.

<button type="button">Play</button>

class VideoPlayer extends React.Component {
  render() {
    return (
      <div className="player">
        <img src="images/house-of-cards.jpg" />
        <PlayButton />
      </div>
    );
  }
}

class PlayButton extends React.Component {
  render() {
    return <button type="button">Play</button>;
  }
}

Data

How do I tell the button if it's a "Play" or "Pause" button?

1-way Data Flow in React

Data flows from parents to children.

Props and State

Props - Data from a parent component.

State - Data owned by the current component.

class VideoPlayer extends React.Component {
  constructor() {
    this.state = { isPlaying: false };
  }

  render() {
    ...
      <PlayButton isPlaying={this.state.isPlaying} />
    ...
  }
}

class PlayButton extends React.Component {
  render() {
    console.log(this.props.isPlaying);
  }
}

Just Describe your Page

If isPlaying is true, write "Pause". If false, write "Play."

Don't worry about what it was before. React handles that for you. Simple!


class PlayButton extends React.Component {
  render() {
    return (
      <button>
        {this.props.isPlaying ? 'Pause' : 'Play'}
      </button>
    );
  }
}

But how can clicking "Play" change the "isPlaying" state in VideoPlayer?

Callbacks

Parents give the children a function to call when they need to pass something back up the tree.


class VideoPlayer extends React.Component {
  // The function to "give" to the child.
  togglePlaying() {
    this.setState({
      isPlaying: !this.state.isPlaying
    });
  }

  render() {
    ...
      <PlayButton
        onClick={this.togglePlaying.bind(this)}
        isPlaying={this.state.isPlaying} />
    ...
  }
}
          

class PlayButton extends React.Component {
  render() {
    return (
      <button onClick={this.props.onClick}>
        {this.props.isPlaying ? 'Pause' : 'Play'}
      </button>
    );
  }
}

React in a Nutshell

  • Break pages into components.
  • Data flows down from parents to children.
  • All data kept in Props and State.
  • Just describe your page ("declarative").
